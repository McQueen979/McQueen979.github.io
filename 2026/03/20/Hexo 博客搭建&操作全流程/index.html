Hexo 博客搭建&操作全流程（含常见问题+避坑指南）

本文结合本人实际操作，整理了Hexo博客从基础配置、文章发布、主题修改到常见问题排查的全流程，重点标记本人频繁出错、踩坑的环节，方便后续快速查阅和操作。

一、前期准备（已完成）

  • 本地环境：Git、Node.js、Hexo 已安装

  • 博客仓库：GitHub 仓库（用户名.github.io）已创建，SSH 连接测试成功（ssh -T git@github.com 能正常返回问候语）

  • 主题：已尝试 Butterfly 主题，最终使用 Papa 主题（配置完成，可本地预览）

  • 工具：已创建 2 个一键操作 BAT 文件（本地预览、发布网站）

二、核心操作步骤（必看，按顺序执行）

1. 文章撰写与格式要求（高频出错点⚠️）

每篇文章必须放在指定文件夹，且顶部必须有标准抬头，否则 Hexo 不识别，网站不显示！

  • 文章存放路径：网站根目录 / source / _posts / 文章名.md（❌ 不可放错文件夹，不可直接放 source 根目录）

  • 文章抬头模板（2种常用，直接复制修改）：

    • 学习版： --- title: 文章标题（不可空） date: 2026-03-20 14:00:00（格式不可错，可修改时间） categories: 学习记录（大分类，可自定义） tags: [Java, 知识点, Hexo]（小标签，方便搜索） --- 正文内容...

    • 生活版： --- title: 文章标题（不可空） date: 2026-03-20 14:00:00（格式不可错） categories: 生活感悟（可自定义） tags: [日常, 成长, 心得]（可自定义） --- 正文内容...

  • ⚠️ 重点坑：忘记加抬头、抬头格式错误（如少写 ---、date 格式错误），会导致文章不显示！

2. 主题修改步骤（高频出错点⚠️）

核心：主题修改后，必须启用主题 + 清理缓存，否则不生效！

  1. 下载主题：将主题文件夹（如 papa）放入 网站根目录 / themes /（⚠️ 文件夹名字必须和配置中一致，不可加后缀，如 papa-master）

  2. 启用主题（最容易忘⚠️）：

    1. 打开 网站根目录 的 _config.yml（不是 themes 文件夹里的主题配置文件）

    2. 找到 theme: landscape，改成 theme: papa（和 themes 里的主题文件夹名完全一致）

    3. 保存文件（Ctrl+S）

  3. 修改主题配置：打开 themes / papa / _config.yml，根据需求修改（如导航栏、背景图、版权信息等）

  4. 本地预览：双击「本地预览.bat」，打开浏览器访问 http://localhost:4000，确认主题修改生效

3. 一键操作 BAT 文件使用（核心工具）

两个 BAT 文件均放在 网站根目录 下，双击即可使用，无需手动输入命令。

（1）本地预览.bat（用于查看修改效果，不上传）

使用说明：双击后，浏览器输入 http://localhost:4000，即可查看文章、主题修改效果，关闭黑窗口则停止预览。

（2）发布网站.bat（用于上传到 GitHub，更新网站）

使用说明：双击后，自动执行「清理缓存→生成网站→上传GitHub」，无需手动输入命令，等待黑窗口显示“发布成功”即可。

4. 网站发布与更新流程（完整步骤）

  1. 撰写文章：在 source/_posts 新建 MD 文件，添加标准抬头，编写正文

  2. 本地预览：双击「本地预览.bat」，确认文章、主题显示正常

  3. 发布更新：双击「发布网站.bat」，等待执行完成（约3-5秒，文章多则稍久）

  4. 查看效果：等待 1-3 分钟，打开浏览器访问 https://用户名.github.io，按 Ctrl+Shift+R 强制刷新（避免缓存坑⚠️）

  5. 删除文章：从 source/_posts 中删除对应 MD 文件，重复步骤 3、4，即可删除网站上的旧文章

三、常见问题排查（本人高频踩坑，重点标记⚠️）

问题1：双击「发布网站.bat」没反应、黑框一闪就没

  • 原因：BAT 脚本缺少 Git 环境路径，或路径错误

  • 解决方法：使用本文中提供的最新版 BAT 脚本（已添加 Git 环境配置），无需修改路径（路径已适配你的电脑）

问题2：主题修改后，发布网站不生效（本地预览正常）（高频坑⚠️）

  • 原因1：未在「网站根目录 _config.yml」中启用主题（忘记改 theme: xxx）

  • 原因2：未清理缓存（Hexo 加载旧缓存，显示旧主题）

  • 原因3：主题文件夹名字和配置中不一致（如文件夹叫 papa-master，配置写 theme: papa）

  • 解决方法：

    • 确认根目录 _config.yml 中 theme: papa（和 themes 文件夹名一致）

    • 删除根目录下的 .deploy_git 和 public 文件夹（旧缓存）

    • 双击「发布网站.bat」，重新发布

    • 浏览器按 Ctrl+Shift+R 强制刷新

问题3：发布网站成功，但浏览器看不到更新（高频坑⚠️）

  • 原因：浏览器缓存顽固（尤其是火狐浏览器），未强制刷新

  • 解决方法：

    • 火狐浏览器：按 Ctrl+Shift+R（强制清空缓存+刷新）

    • 其他浏览器（如Chrome）：按Ctrl+F5 强制刷新

    • 若仍不显示：等待 3 分钟（GitHub Pages 同步有延迟），再次强制刷新

问题4：发布时出现「JSON 解析错误」（EJSONPARSE）

  • 原因：修改 package.json 时，语法错误（少加逗号、括号不匹配）

  • 解决方法：

    • 打开根目录 package.json，找到 "scripts" 段落

    • 复制以下正确内容，覆盖原有 scripts 段落： "scripts": { "build": "hexo generate", "clean": "hexo clean", "server": "hexo server", "deploy": "hexo clean && hexo g && hexo d" },

    • 保存文件，重新双击「发布网站.bat」

问题5：发布时出现「Theme 脚本错误」（require(…) is not a function）

  • 原因：主题版本与 Hexo 版本不兼容（如 Butterfly 最新版不兼容旧版 Hexo）

  • 解决方法（二选一）：

    • 升级 Hexo 到最新版（推荐，适配新主题）： npm install hexo@latest -g npm install hexo@latest --save npm install hexo-renderer-pug hexo-renderer-stylus --save npm install

    • 更换兼容的主题版本（如 Butterfly 4.5.0 稳定版）

问题6：发布时出现「GitHub 连接失败」（Failed to connect to github.com port 443）

  • 原因：使用 HTTPS 格式上传，国内网络不稳定；或 SSH 未配置成功

  • 解决方法：

    • 打开根目录 _config.yml，修改 deploy 为 SSH 格式：

    • deploy: type: git repo: git@github.com:用户名/用户名.github.io.git branch: main

    • 删除根目录 .deploy_git文件夹，重新发布

    • 若仍失败：手动上传（保险方案）

      1. 双击「本地预览.bat」，等待生成 public 文件夹

      2. 打开 public 文件夹，全选所有文件

      3. 打开 GitHub 仓库，直接上传覆盖所有文件，提交即可

问题7：文章撰写后，网站不显示

  • 原因1：文章未放在 source/_posts 文件夹

  • 原因2：文章顶部没有标准抬头，或抬头格式错误

  • 原因3：未清理缓存，未重新发布

  • 解决方法：

    • 确认文章路径和抬头格式正确

    • 双击「发布网站.bat」重新发布

    • 浏览器强制刷新

问题8：网站必须挂梯子才能访问（高频坑⚠️）

  • 原因：核心是网站部署在 GitHub Pages 上，而 GitHub 属于国外服务器，国内网络访问时会出现 DNS 解析污染、CDN 节点调度异常 等问题，并非被“墙”，而是国内网络与国外服务器的连接不稳定，导致不挂梯子无法正常加载页面或加载极慢。具体来说，国内运营商的 DNS 可能会把 GitHub 相关域名解析到错误的 IP 地址，或者 TLS 握手失败，导致浏览器无法正常连接到你的博客服务器。

  • 补充说明：这不是你操作的问题，所有部署在 GitHub Pages 上的个人博客，国内访问都可能遇到这个情况（不同网络环境差异，部分宽带偶尔能直接访问，但不稳定），和你用的 Papa 主题、Hexo 配置无关。

  • 解决方法（按简单到复杂排序，优先试前2种，无需复杂操作）： 修改本地 DNS（最推荐，无需装软件，2分钟生效）： 打开电脑「网络和共享中心」→ 找到当前连接的网络（WiFi/有线）→ 右键「属性」→ 双击「Internet 协议版本 4 (TCP/IPv4)」

  • 选择「使用下面的 DNS 服务器地址」，输入： 首选 DNS：1.1.1.1（Cloudflare 公共 DNS，稳定） 备用 DNS：1.0.0.1

  • 保存后，关闭浏览器重新打开，无需挂梯子，尝试访问你的博客（部分网络需等待1-2分钟生效）。

使用 GitHub 镜像网站访问（无需修改任何设置）： 将你博客的地址 https://用户名.github.io 中的 github.io 替换为 hub.fastgit.org，即 https://用户名.hub.fastgit.org

这个镜像网站与 GitHub 内容完全同步，国内访问速度快，无需挂梯子，适合自己查看或分享给他人。

进阶方法（需简单配置，更稳定）： 下载 SwitchHosts 工具（免费），通过工具修改本地 hosts 文件，将 GitHub 相关域名与正确的 IP 绑定，解决 DNS 解析错误问题，具体操作可搜索“SwitchHosts 配置 GitHub”，有现成的 IP 列表可复制粘贴。

⚠️ 重点提醒：挂梯子访问只是临时解决方案，优先尝试修改 DNS 或使用镜像网站，操作简单且能长期解决问题；如果是分享给国内朋友，直接发镜像地址即可，对方无需挂梯子也能正常访问。

四、重点避坑总结（必记）

  • ⚠️ 文章必须放 source/_posts，必须加标准抬头，否则不显示

  • ⚠️ 主题修改后，必须在「根目录 _config.yml」启用，且清理缓存，否则不生效

  • ⚠️ 发布成功后，浏览器必须强制刷新（火狐 Ctrl+Shift+R），否则看不到更新

  • ⚠️ 发布失败优先检查：deploy 是 SSH 格式、删除 .deploy_git 文件夹

  • ⚠️ 本地预览正常 = 配置、主题、文章都没问题，不更新就是缓存或上传问题

  • ✅ 两个 BAT 文件无需修改，双击即可使用，发布前确保根目录 _config.yml 中 theme 正确

五、常用命令（备用，无需手动输入，用BAT 已集成）

  • 清理缓存：hexo clean

  • 生成网站：hexo g（g = generate）

  • 本地预览：hexo s（s = server）

  • 发布网站：hexo d（d = deploy）

  • 一键发布（备用）：npm run deploy（需在 package.json 中配置脚本）
